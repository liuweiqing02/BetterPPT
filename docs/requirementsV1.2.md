# 《requirementsV1.2》

## 1. 项目背景

在“文档转 PPT”场景中，常见问题包括：

- 内容表达生硬，难以形成汇报叙事。
- 图表与关键信息在转换过程中丢失。
- 视觉风格与企业模板不一致，需大量人工返工。

BetterPPT V1.1 已实现：上传、任务流转、基础模板分析、LLM 页级大纲规划、PPT 导出。

V1.1 的核心缺口是：导出链路本质仍偏向“新建空白页后填充标题/要点”，未形成“参考 PPT 内元素级增删改”的生成机制，导致结构语义复用不足、风格保真度不稳定、人工返工成本较高。

V1.2 目标是在 V1.1 基础上升级为“模板资产化 + 槽位映射填充 + 元素级编辑”，将参考 PPT 从“风格参考”升级为“可编辑模板资产”。

## 2. 项目目标

- 用户必须上传 `PDF 文档` 与 `参考 PPT`，并选择输出长度挡位（精炼/适中/详细）。
- 系统自动决定最终页数、章节结构与页面内容分配。
- 对长文档支持可选提示词，结合 RAG 进行内容重点控制。
- 参考 PPT 中关键视觉与结构元素必须保留并复用，不仅是风格参考。
- 先沉淀“分类模板页/结构化内容页模板”，再进行内容槽位填充。
- 填充内容来源包括：文本（LLM 提炼）、文档图像、文档表格。
- 输出风格与参考 PPT 高相似，结果为可编辑 `.pptx` 并支持预览与下载。

## 3. V1.2 相比 V1.1 的变更清单

### 3.1 新增

- 新增“模板资产化与元素保留策略”能力。
- 新增“内容槽位映射与填充规则（文本/图像/表格）”能力。
- 新增“结构化模板页资产库”概念与复用流程。
- 新增量化验收指标：布局偏移、尺寸偏差、样式保真、槽位匹配率、自动修正与回退成功率、可编辑性。
- 新增范围边界章节（MVP 必做/后续可做/明确不做）。
- 新增细化风险与回退策略章节。

### 3.2 修改

- 生成主链路从“空白页写内容”为主，升级为“模板页复制 + 元素级增删改”为主。
- 模板分析产物从“聚类结果为主”升级为“页面分类 + 槽位 schema + 样式 token + 母版语义”。
- 质量评估口径从主观抽检为主，升级为“自动量化指标 + 人工抽检”。
- 错误处理从任务级失败为主，升级为“步骤级自修正 + 可回退 + 可审计”。

### 3.3 删除

- 删除“默认以新建空白页为主要导出策略”的产品口径。
- 删除“参考 PPT 仅用于风格提示、不参与结构约束”的策略。
- 删除“图像/表格仅转文字摘要即可通过”的验收口径。

## 4. 用户价值与业务价值

### 4.1 用户价值

- 生成结果更贴近企业模板，减少手工调版。
- 文档图像和表格信息保留更完整，可读性与可信度更高。
- 导出结果可直接编辑，支持汇报前快速微调。

### 4.2 业务价值

- 提高一次交付通过率，降低运营与人工润色成本。
- 模板资产可沉淀复用，降低后续任务边际生成成本。
- 可量化质量指标可用于研发迭代、测试回归与 SLA 管理。

## 5. 术语定义

- 文档：用户上传的源 PDF 文件。
- 参考 PPT：用户上传的风格模板 PPTX 文件。
- 长度挡位：输出密度配置，包含精炼、适中、详细三档；后端枚举值为 `concise|balanced|detailed`。
- 编辑式生成：将“生成 PPT”转化为“对模板 PPT 的批量增删改”。
- 结构化页面：封面、目录、章节页、结论页、对比页等具有固定布局功能的页面类型。
- 模板资产化：将参考 PPT 解析并沉淀为可复用的模板页面资产（含结构语义、槽位 schema、样式 token）。
- 槽位：页面内承载内容的目标区域，分文本槽位、图像槽位、表格槽位。
- 样式 token：可复用的视觉样式参数，如字体、字号、色值、行距、对齐、间距等。

## 6. 用户角色与使用流程

### 6.1 用户角色

- 业务汇报人员
- 咨询/研究人员
- 产品与运营人员

### 6.2 主流程

1. 上传 PDF 与参考 PPT（必填）。
2. 选择长度挡位（必选：精炼/适中/详细）。
3. 可选输入提示词（长文档场景下指定重点）。
4. 提交任务并查看处理进度。
5. 系统执行模板资产化、内容规划、槽位映射、元素级填充与修正。
6. 预览生成结果并下载 PPT。

## 7. 功能需求

### 7.1 任务创建与校验

- 前端必须校验两类文件是否齐全，不齐全不可提交。
- 文件格式限制：
  - 文档：PDF
  - 模板：PPT/PPTX（输出统一为 PPTX）
- 可配置文件大小上限、页数上限、并发提交限制。
- 后端二次校验文件角色：
  - `source_file_id` 必须为 `pdf_source`
  - `reference_file_id` 必须为 `ppt_reference`

### 7.2 文档理解与内容规划

- 抽取文档结构（标题层级、段落、图表、关键术语）。
- 基于长度挡位自动规划页数与内容密度。
- 长文档支持 RAG：
  - 根据用户提示词做检索增强。
  - 提升指定主题内容在输出中的权重。

### 7.3 长度挡位规则（MVP 可验收口径）

以 20-60 页输入 PDF 为基准，目标页数范围如下：

- `concise`（精炼）：8-12 页，强调结论与行动项。
- `balanced`（适中）：13-20 页，信息与可读性平衡。
- `detailed`（详细）：21-30 页，覆盖更多细节与论据。

跨挡位约束：

- 同一输入文档下，`page_count(concise) < page_count(balanced) < page_count(detailed)` 必须成立。
- 文本密度（内容页平均正文字符数）应随挡位递增。
- 关键章节覆盖率（问题/分析/结论）三档均不低于 90%。

### 7.4 提示词与 RAG 行为约定

- `user_prompt` 为空，`rag_enabled=false`：默认摘要与规划流程。
- `user_prompt` 非空，`rag_enabled=false`：提示词仅作为规划偏好，不触发检索增强。
- `user_prompt` 非空，`rag_enabled=true`：提示词参与检索增强并提升主题权重。
- `user_prompt` 为空，`rag_enabled=true`：系统自动从文档关键词构造检索查询。

### 7.5 多模态模板分析

- 使用 ViT 提取参考 PPT 各页 embedding。
- 采用层次聚类得到模板页面类型集合。
- 结合统一 LLM API（默认 `gpt-4.1-mini`）抽取每类页面结构化信息：
  - 页面功能（封面、目录、章节、对比、总结等）
  - 布局 schema（文本区、图片区、表格区、占比关系）
- 将结构 schema 作为后续生成约束，提高视觉迁移保真度。

### 7.6 模板资产化与元素保留策略

- 参考 PPT 不再仅作为“风格参考”，而是“可编辑模板资产”。
- 必须保留并可复用以下元素：
  - 页面结构与分类（封面/目录/章节/内容/总结等）
  - 文本框位置、尺寸、层级、样式
  - 为美观服务的图像与装饰元素
  - 主题色、字体、母版/版式语义
- 系统先沉淀“分类模板页/结构化内容页模板”，再进行内容填充。
- 模板页复用优先级：同类模板页 > 相近模板页 > 兜底通用模板页。

### 7.7 内容槽位映射与填充规则（文本/图像/表格）

- 文本填充：
  - LLM 输出标题、要点、说明文本并映射到文本槽位。
  - 超长文本按“拆页优先、摘要次之、缩放最后”策略处理。
- 图像填充：
  - 优先提取文档原生图像并映射至图像槽位。
  - 图像比例不一致时执行等比缩放、智能裁剪或留白对齐。
- 表格填充：
  - 优先还原为原生 PPT 表格对象。
  - 复杂表格无法稳定还原时，降级为图片 + 标题/说明。
- 映射约束：
  - 不覆盖关键装饰元素与品牌识别元素。
  - 不突破母版安全边距。
  - 映射失败需触发自修正与回退流程。

### 7.8 编辑式生成引擎

- 在参考模板基础上执行元素级批量增删改：
  - 替换文本内容
  - 插入/替换图片
  - 插入/替换表格
  - 调整图文布局
  - 新增或删除页面
- 严禁将整页栅格化为不可编辑图片作为默认路径。

### 7.9 自我修正与失败回退

- 检测维度：溢出、遮挡、空白异常、错位、对齐异常。
- 自动修正动作：重排、换行、缩放、换槽位、拆页。
- 回退策略（按顺序）：
  1. 同类模板页替换
  2. 相近模板页替换
  3. 简化布局模板兜底
- 回退后仍失败：标记失败原因并保留可审计日志。

### 7.10 任务管理与结果交付

- 提供任务状态：`created`、`validating`、`queued`、`running`、`succeeded`、`failed`、`canceled`。
- 失败支持错误原因查看与重试。
- 成功任务支持在线预览与文件下载。
- 预览接口仅允许 `succeeded` 任务调用，返回 `slides` 与 `expires_in`。
- 生成过程记录模板匹配、槽位映射、修正和回退事件。

## 8. 用户端界面与交互需求

### 8.1 页面范围（MVP）

- 任务创建页：上传 PDF 与参考 PPT、选择长度挡位、可选提示词输入、提交任务。
- 任务列表页：展示任务状态、创建时间、最后更新时间、失败原因摘要、快捷操作（查看详情/重试）。
- 任务详情页（进度 + 日志）：展示状态机进度、当前步骤、关键事件日志、回退/修正记录。
- 结果预览与下载页：展示预览缩略图列表（slides），支持下载最终 PPT。
- 失败重试入口：列表页与详情页均提供重试入口，并显示重试次数与最近失败原因。

### 8.2 交互要求

- 表单校验：
  - 必填项校验：未上传 PDF 或参考 PPT 时禁止提交。
  - 格式校验：仅允许 PDF、PPT/PPTX；超限文件给出明确提示。
  - 联动校验：长度挡位未选中时提交按钮置灰并提示。
- 上传进度：
  - 显示单文件上传进度百分比与总进度。
  - 上传失败时支持单文件重传，不要求整单重提。
- 轮询与状态提示：
  - 任务创建成功后自动轮询任务状态，默认轮询间隔 3 秒（可配置）。
  - 状态变化需可视化提示（处理中、修正中、回退中、已完成、失败）。
  - “回退中”判定规则：以任务详情字段 `fallback_state=running` 为准；当 `fallback_state in (succeeded,failed)` 或任务进入终态时结束展示。
- 失败可恢复：
  - 对 `failed` 任务展示“重试”按钮与失败原因。
  - 对可回退阶段失败需显示“系统已尝试回退/回退失败”的状态文案。
- 上传限制配置：
  - 文件大小与类型限制由后端配置接口下发，前端按配置前置拦截，不允许硬编码阈值。
- 空状态与错误提示文案：
  - 任务列表空状态：`暂无任务，上传 PDF 和参考 PPT 开始生成。`
  - 预览空状态：`结果尚未生成，请稍后刷新。`
  - 网络错误：`网络异常，请检查后重试。`
  - 系统繁忙：`系统繁忙，请稍后再试。`

### 8.3 体验要求

- 桌面适配：
  - 以桌面端为主，最小支持分辨率 `1280 x 720`。
  - 在 `1366 x 768` 与 `1920 x 1080` 下保证核心区域无错位、无遮挡、无横向滚动条。
- 首屏与关键操作响应时间（前端可观测口径）：
  - 首屏可交互时间（任务创建页）：P95 <= 2.0 秒。
  - 提交任务后拿到受理结果：P95 <= 1.5 秒（不含生成耗时）。
  - 任务列表与详情切换响应：P95 <= 1.0 秒。
  - 结果预览首批缩略图可见：P95 <= 2.5 秒（任务已成功前提下）。
- 统计口径（用于验收与争议裁决）：
  - 统计窗口：按自然周统计；发布后前 7 天可按日观察但不作为最终 SLA 结论。
  - 样本量下限：单指标有效样本 `N >= 200`。
  - 冷/热启动口径分开统计：SLA 以热启动口径为准，冷启动单独看板展示。
  - 弱网排除：`effectiveType=2g/slow-2g` 或丢包率异常样本不计入 SLA，但保留监控。

### 8.4 可验收标准（界面与交互）

- MVP 四个页面（创建/列表/详情/预览下载）均可独立访问并完成主流程；失败重试作为列表页与详情页的可达操作入口验收。
- 表单校验、上传进度、轮询状态、失败重试、空状态与错误文案均有对应 UI 呈现。
- 在桌面端目标分辨率下完成 20 次回归操作，无阻断级 UI 缺陷。
- 关键交互响应时间满足 8.3 目标，且可通过埋点或前端监控复核。

## 9. 非功能需求

### 9.1 性能与容量（MVP）

- 普通任务端到端耗时目标：P50 <= 6 分钟，P95 <= 12 分钟。
- 与 V1.1 的 5/10 目标相比，V1.2 调整为 6/12，原因是新增模板资产化、槽位映射、质量门控与回退链路，执行路径更长。
- 容量假设：输入文档 20-60 页、单租户并发 3、工作节点按横向扩容策略部署。
- 系统可用性目标：月度 >= 99.5%。
- 任务成功率目标（非用户取消）：>= 95%。
- 单租户默认并发提交阈值：3（可配置）。

### 9.2 稳定性

- 任务失败可恢复，支持幂等重试。
- 后端关键流程具备超时与降级策略。
- Worker 支持断点续跑（按步骤跳过已完成节点）。

### 9.3 安全与治理

- 文件隔离存储，按任务与用户进行访问控制。
- 上传文件执行白名单与恶意文件扫描（扩展名 + MIME + 扫描结果）。
- 下载链接使用短时效签名。
- 默认留存策略（可配置）：
  - 源文件：7 天
  - 结果文件：30 天
  - 任务审计日志：180 天
- 支持用户发起删除与到期自动清理。

### 9.4 可观测性

- 记录任务链路日志与关键耗时指标。
- 提供错误分类统计，支持后续模型与策略优化。
- 记录任务事件流用于前端进度展示与问题回溯。

## 10. 技术方案约束

### 10.1 技术栈

- 主要语言：Python
- 架构：前后端分离
- 数据库：MySQL + Redis
- 远程仓库：<https://github.com/liuweiqing02/BetterPPT.git>

### 10.2 运行与开发要求

- 本地运行必须使用虚拟环境（venv/conda）。
- 提供 `.env.example`，最少包含：
  - MySQL 连接配置
  - Redis 连接配置
  - 模型服务配置（必须包含以下变量）：
    - `LLM_API_BASE`（默认：`https://api.openai.com/v1`）
    - `LLM_API_KEY`
    - `LLM_MODEL`（默认：`gpt-4.1-mini`）
  - 对象存储配置（如有）

### 10.3 Agent Worker 技术约束

- 所有核心 Agent 必须由 LLM 驱动，并通过统一 API 客户端调用（OpenAI 兼容协议）。
- 本地模型仅用于辅助能力，不替代核心推理链路：
  - PPT 视觉 embedding（如 ViT）
  - OCR/版面检测（如需要）
  - 轻量规则分类器
- Worker 内部步骤最少包含以下 Agent，并要求结构化输入输出：
  - `DocumentParseAgent`
  - `RagRetrieveAgent`
  - `OutlinePlanAgent`
  - `LayoutMapAgent`
  - `SlideGenerateAgent`
  - `SelfCorrectAgent`
- 所有 Agent 输出必须可追踪（入库 `task_steps.output_json` 与 `task_events`）。

## 11. 系统架构（逻辑）

- 前端：上传、参数选择、进度与结果展示。
- 后端 API：任务受理、鉴权、状态查询、结果预览地址下发。
- Agent 编排层：
  - 文档解析 Agent
  - 检索增强 Agent（RAG）
  - 大纲规划 Agent
  - 版式映射 Agent
  - 质量修正 Agent
- 存储层：
  - MySQL：用户、任务、文件元数据、模板分析、步骤与事件日志
  - Redis：任务队列、缓存、分布式锁、进度状态

## 12. 数据模型（与 TDD 对齐）

### 12.1 MySQL 核心表

- `users`：用户信息
- `files`：上传与产出文件元数据
- `tasks`：任务主表（状态、挡位、耗时、错误码）
- `task_steps`：任务步骤执行记录
- `task_events`：任务事件流
- `template_profiles`：模板聚类分析结果
- `template_page_schemas`：模板页面结构化 schema

### 12.2 Redis 使用

- `stream:tasks:pending`：任务队列（Stream + Consumer Group）
- `lock:task:{task_no}`：任务级分布式锁
- `task:progress:{task_no}`：任务状态短期缓存
- `task:events:{task_no}`：最近 N 条事件缓存
- `rate_limit:create_task:{user_id}`：限流计数

## 13. 范围边界

### 13.1 MVP 必做

- 模板资产化（页面分类 + 槽位 schema + 样式 token）。
- 模板页复制 + 元素级文本/图像/表格填充主链路。
- 自动修正（溢出/遮挡/错位）与至少一层回退机制。
- 可编辑 PPT 导出与可观测日志。
- 量化验收指标与自动化测试样例。

### 13.2 后续版本可做

- 图表语义重建（从表格自动生成柱状/折线图并套用模板样式）。
- 跨模板混合生成与企业模板推荐。
- 多文档融合生成、跨语言排版优化。
- 设计偏好学习与用户个性化模板资产库。

### 13.3 明确不做（V1.2）

- 不做复杂动画迁移与触发器还原。
- 不做 SmartArt 全量语义重建。
- 不做扫描 PDF 的高精度 OCR 纠错闭环（仅基础支持）。
- 不做手写批注、公式排版的高保真重建。

## 14. 验收标准（MVP）

- 必填项校验正确：未上传 PDF 或参考 PPT 不可提交。
- 三档长度挡位生效：同文档下页数单调递增，且落入对应挡位范围。
- 长文档提示词可影响内容取舍；当 `rag_enabled=true` 时可观察到检索命中差异。
- 同一参考模板下，生成结果页面结构语义保留率 >= 90%。
- 文本框布局偏移 <= 3%，尺寸偏差 <= 8%。
- 样式保真度：字体族一致率 >= 90%，字号偏差 <= 2pt，主色命中率 >= 90%。
- 槽位匹配成功率：
  - 文本槽位 >= 95%
  - 图像槽位 >= 90%
  - 表格槽位 >= 85%
- 溢出/遮挡/错位自动修正成功率 >= 95%，修正失败后的回退产出成功率 >= 98%。
- 最终可编辑性：文本可编辑对象占比 >= 95%，整页锁死/位图化页面占比 <= 2%。
- 任务状态可追踪，失败可重试，成功可预览可下载。
- 核心接口具备契约测试，状态机流转具备单元/集成测试。

### 14.1 指标计算口径与版本化（V1.2.2）

- 指标版本字段：`metric_version`，默认 `v1.0`，口径变化必须升级版本。
- 统计粒度：
  - 布局/样式指标按对象粒度统计。
  - 槽位匹配指标按槽位粒度统计。
  - 修正/回退指标按任务事件粒度统计。
- 样本页范围：
  - 默认评估内容页与章节页。
  - 默认排除封面与目录页（排除范围需写入质量报告）。
- 分母定义：
  - 文本/图像/表格槽位匹配率分母分别为对应槽位总数。
  - 自动修正成功率分母为自动修正触发次数。
  - 回退成功率分母为回退触发次数。
  - 可编辑性分母为文本对象总数。
- 测试复算要求：测试团队可基于任务日志与导出文件独立复算并得到一致结果。

## 16. 风险与应对

- 风险：模板解析失败导致无法形成可复用资产。  
  应对：启用基础槽位提取与相近模板页回退；保留失败页日志用于复跑。

- 风险：内容槽位不匹配导致塞不下或语义错配。  
  应对：拆页优先，摘要压缩次之，再执行模板替换和简化布局回退。

- 风险：图表/表格还原失败导致信息损失。  
  应对：优先原生表格，失败降级为图片 + 标题说明 + 数据摘要文本兜底。

- 风险：版面溢出、遮挡、错位影响可读性。  
  应对：自动检测与重排，超过阈值触发模板回退与样式轻降级。

- 风险：模型调用波动导致任务不稳定。  
  应对：关键步骤超时熔断、重试、温度控制、候选打分与规则兜底。

- 风险：回退链路失效导致任务失败率上升。  
  应对：对回退路径单独建立回归测试与压测基线，按周监控并修复。
